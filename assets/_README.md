# assets/ — 회사 양식(템플릿) 폴더

문서 생산 스튜디오는 **콘텐츠를 회사 양식에 흘려넣는** 방식으로 산출물을 만듭니다.
(빈칸을 채우는 게 아니라, 양식이 정의한 **스타일/레이아웃** 위에 본문이 들어갑니다.)

여기에 양식 파일을 그대로 두면 **자동 적용**됩니다. 없으면 내장 기본 서식으로 생성됩니다.

| 파일명 | 용도 | 스펙 |
|---|---|---|
| `hwpx_template.hwpx` | 한글 보고서 양식 | 아래 §HWPX |
| `pptx_template.pptx` | 발표자료(슬라이드) 양식 | 아래 §PPTX |
| `docx_template.docx` | Word 보고서 양식(선택) | 명명 스타일 상속 |

> 레시피(YAML)의 `output.template` 로 파일별 다른 양식을 지정할 수도 있습니다.
> 경로는 이 `assets/` 폴더 기준 상대경로 또는 프로젝트 루트 기준입니다.

## HWPX 양식 만들기 (요약)
- A4 세로, 한글의 **스타일(F6)** 에 다음 이름을 **정확히** 정의: `제목`, `제목 1`, `제목 2`,
  `제목 3`, `본문`, `목록`, `강조`, `인용`, `코드`.
- 우리 코드가 Markdown `#/##/###` → `제목 1/2/3`, 일반 단락 → `본문`, 불릿 → `목록` 으로 매핑합니다.
- 표지에 `_제목_입력_` 텍스트를 두면 보고서 제목으로 치환됩니다.
- 자세한 사양: [docs/studio/05_양식(HWPX·PPTX)_만들기.md](../docs/studio/05_양식_만들기.md)

## PPTX 양식 만들기 (요약)
- 16:9, 슬라이드 마스터에 **Layout #0(제목+부제)**, **Layout #1(제목+본문)** 정의.
- Layout #1 의 본문 placeholder 에 불릿 3~5개가 자동으로 들어갑니다.

## HWPX 생성 두 가지 경로
1. **한컴 OLE**(권장·정식): Windows + 한컴 한글 + `pywin32` → 양식을 열고 본문 주입.
2. **ZIP 폴백**: 한글이 없을 때, 위 `hwpx_template.hwpx` 의 스타일을 재사용해 본문만 생성.
   둘 다 불가하면 Markdown(.md) 으로 제공합니다.
