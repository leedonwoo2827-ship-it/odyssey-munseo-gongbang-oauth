# PPTX 템플릿 규칙 (디자인 담당자용)

문서공방의 PPTX 생성 엔진(`services/studio/generators/pptx_gen.py`)이 회사 양식
(`assets/pptx_template.pptx`)을 자동으로 채웁니다. 엔진은 **placeholder 번호가 아니라
'유형(type)'으로** 자리를 찾으므로, 아래 규칙만 지키면 어떤 디자인이든 깔끔하게 출력됩니다.

> 현재 회사 템플릿은 이 규칙에 맞아 **그대로 잘 동작합니다(재제작 불필요)**. 이 문서는
> 앞으로 템플릿을 새로 만들거나 수정할 때를 위한 안내입니다.

---

## 1. 꼭 필요한 레이아웃 2종

| 레이아웃 | 이름에 포함(권장) | 있어야 할 placeholder |
|---|---|---|
| 표지 | `표지` 또는 `제목 슬라이드` | **제목(TITLE/CENTER_TITLE)** 1개 + **부제(SUBTITLE 또는 BODY)** 1개 |
| 본문 | `기본본문` 또는 `제목 및 내용` | **제목(TITLE)** 1개 + **본문(BODY/OBJECT)** 1개 |

- 엔진은 위 이름을 우선 찾고, 없으면 placeholder 유형으로 자동 선택합니다.
- placeholder **번호(idx)는 아무 값이어도 됩니다**(예: 102, 105). 유형만 맞으면 됩니다.

## 2. 엔진이 자동으로 하는 일 (디자이너가 신경 안 써도 됨)

- 표지 = 제목 + 부제 채움 / 본문 = 제목 + 본문(불릿) 채움.
- **채우지 않은 본문·제목 placeholder의 안내문구는 자동으로 비웁니다.**
  → 그러니 "내용을 입력하세요" 같은 **안내문은 반드시 placeholder의 prompt 텍스트로** 두세요
  (일반 텍스트 상자로 두면 안 지워지고 화면에 박힙니다).
- 폰트 크기·줄바꿈·자동맞춤(autofit)은 엔진이 상한을 덮어써 **글자 폭주를 막습니다**.
  → 디자이너는 placeholder의 **위치·크기**만 잘 잡으면 됩니다(폰트는 엔진이 조정).
- 템플릿에 샘플 슬라이드를 넣어둬도 됩니다 → 생성 시 자동 제거되고 새 슬라이드만 남습니다.

## 3. 브랜딩(로고·푸터 등) 처리

- 마스터/레이아웃에 **일반 도형·텍스트로 넣은 것**(예: 로고 바, 푸터 문구)은
  **모든 슬라이드에 그대로 표시**됩니다. → 항상 보이길 원하는 브랜딩만 이렇게 두세요.
- 반대로, **채워질 자리**(제목/부제/본문/날짜)는 반드시 **placeholder**로 만드세요.
- 현재 회사 템플릿의 `PROJECT / PROPOSAL TEMPLATE`, `CONFIDENTIAL`, `LOGO`는 마스터 고정
  텍스트라 모든 장에 나옵니다. 빼고 싶으면 PowerPoint에서 **보기 → 슬라이드 마스터** 로
  들어가 해당 텍스트를 지우면 됩니다(코드 수정 불필요).

## 4. 빠른 점검 체크리스트

- [ ] 표지 레이아웃에 제목 + 부제 placeholder가 있다.
- [ ] 본문 레이아웃에 제목 + 본문 placeholder가 있다.
- [ ] 안내문구는 전부 placeholder의 prompt 텍스트다(일반 텍스트 상자 아님).
- [ ] 항상 보여야 할 브랜딩만 마스터에 일반 도형/텍스트로 두었다.
- [ ] 16:9 (와이드) 크기.

> 교체 방법: 새 양식을 `assets/pptx_template.pptx` 로 덮어쓰면 끝(파일명 동일).
> 특정 레시피만 다른 양식을 쓰려면 레시피 YAML 의 `output.template:` 에 경로를 지정.
