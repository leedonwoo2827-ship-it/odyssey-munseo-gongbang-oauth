# 05 · 회사 양식(HWPX·PPTX) 만들기

이 도구는 **빈칸을 채우는 방식이 아닙니다.** 양식이 정의한 **스타일/레이아웃**(디자인) 위에
LLM이 만든 **본문 콘텐츠가 흘러 들어가는** 방식입니다. 그래서 양식은 "내용"이 아니라 "틀"만
정의하면 됩니다.

양식 파일은 `assets/` 폴더에 두면 자동 적용됩니다(없으면 내장 기본 서식).

| 파일 | 용도 |
|---|---|
| `assets/hwpx_template.hwpx` | 한글 보고서 양식 |
| `assets/pptx_template.pptx` | 발표자료(슬라이드) 양식 |
| `assets/docx_template.docx` | Word 보고서 양식(선택) |

> 레시피별로 다른 양식을 쓰려면 `output.template` 에 파일을 지정하세요.

---

## HWPX 양식 (한글)
A4 세로 문서를 만들고, 한글의 **스타일 기능(F6)** 에 아래 이름을 **정확히** 정의합니다.
우리 코드가 Markdown 을 이 이름의 스타일로 매핑합니다.

| 스타일 이름 | 매핑되는 내용 | 권장 서식 |
|---|---|---|
| `제목` | 표지 큰 제목 | 24~28pt 굵게, 메인 컬러 |
| `제목 1` | `#` (장) | 18pt 굵게 |
| `제목 2` | `##` (절) | 14pt 굵게 |
| `제목 3` | `###` (항) | 12pt 굵게 |
| `본문` | 일반 단락 | 10~11pt, 줄간격 160% |
| `목록` | 불릿 `-` | 들여쓰기 |
| `강조` | `**굵게**` | 포인트 컬러 굵게 |
| `인용` | `>` | 이탤릭/좌측 라인 |
| `코드` | 인라인 코드 | 고정폭(D2Coding) |

- 표지에 `_제목_입력_` 텍스트를 넣어두면 보고서 제목으로 치환됩니다.
- 머리말/꼬리말/로고는 양식에 박아두면 그대로 유지됩니다(코드가 건드리지 않음).
- 완료 후 `assets/hwpx_template.hwpx` 로 저장 → 앱 재시작.

### HWPX 생성 두 경로 (자동 선택)
1. **한컴 OLE**(권장): Windows + 한컴 한글 + `pywin32`. 양식을 열고 본문을 주입해 `.hwpx` 저장.
2. **ZIP 폴백**: 한글이 없을 때, 위 양식의 **스타일을 재사용**해 본문 섹션만 새로 만들어 `.hwpx` 생성.
3. 둘 다 불가하면 **DOCX/Markdown** 으로 대체 제공(경고로 안내).

> 텍스트 길이가 바뀌면 줄 위치가 약간 어긋날 수 있습니다(한글에서 한 번 열면 자동 재배치).

## PPTX 양식 (슬라이드)
16:9 와이드로 만들고, 슬라이드 마스터에 **두 레이아웃**을 정의합니다.

- **Layout #0 (표지)**: `Title`(발표 제목) + `Subtitle`(부제) placeholder 2개.
- **Layout #1 (본문)**: `Title`(슬라이드 제목) + `Content`(본문) placeholder. 본문에는 **불릿 3~5개**가 자동으로 채워집니다. (다단 아님, 단일 영역)

- 로고/배경/푸터/페이지번호는 마스터에 박아두면 유지됩니다.
- 완료 후 `assets/pptx_template.pptx` 로 저장 → 앱 재시작.

---

## DOCX 양식 (선택)
`assets/docx_template.docx` 에 `제목 1/2/3`, `본문`, `목록` 등 명명 스타일을 정의해 두면
HWPX와 동일한 매핑으로 상속됩니다. 없으면 Word 기본 스타일(Heading 1~3, Normal)로 생성됩니다.

## 검증 체크리스트
- [ ] 파일명/위치가 정확한가 (`assets/…_template.…`).
- [ ] HWPX: 9개 스타일 이름이 정확히 정의됐는가.
- [ ] PPTX: 16:9 + 두 레이아웃에 placeholder가 있는가.
- [ ] 앱 재시작 후 생성물에 디자인이 적용되는가.
