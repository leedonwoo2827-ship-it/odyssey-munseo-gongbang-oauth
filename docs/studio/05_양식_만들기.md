# 05 · 회사 양식(HWPX·PPTX) 만들기

이 도구는 **빈칸을 채우는 방식이 아닙니다.** 양식이 정의한 **스타일/레이아웃**(디자인) 위에
LLM이 만든 **본문 콘텐츠가 흘러 들어가는** 방식입니다. 그래서 양식은 "내용"이 아니라 "틀"만
정의하면 됩니다.

양식 파일은 `assets/` 폴더에 두면 자동 적용됩니다(없으면 내장 기본 서식).

| 파일 | 용도 |
|---|---|
| `assets/hwpx_template.hwpx` | 한글 보고서 양식 |
| `assets/pptx_template.pptx` | 발표자료(슬라이드) 양식 |
| `assets/docx_template.docx` | Word 보고서 양식(선택) |

> 레시피별로 다른 양식을 쓰려면 `output.template` 에 파일을 지정하세요.

---

## HWPX 양식 (한글)
A4 세로 문서를 만들고, 한글의 **스타일 기능(F6)** 에 아래 이름을 **정확히**(공백·숫자까지) 정의합니다.
우리 코드가 Markdown 을 이 이름의 스타일로 매핑합니다.

| 스타일 이름 | 매핑되는 내용 | 권장 서식 |
|---|---|---|
| `제목` | 표지 큰 제목 | 24~28pt 굵게, 메인 컬러 |
| `제목 1` | `#` (장) | 18pt 굵게 |
| `제목 2` | `##` (절) | 14pt 굵게 |
| `제목 3` | `###` (항) | 12pt 굵게 |
| `본문` | **글머리표 없는** 일반 단락 | 10~11pt, 줄간격 160%, 글머리표 없음 |
| `동그라미` | **1단계** 항목(최상위 `-`) → **○** | 본문 + 글머리표 **○**, 들여쓰기 1수준 |
| `마이너스` | **2단계** 항목(중첩 `-`) → **−** | 본문 + 글머리표 **−**, 들여쓰기 2수준 |
| `강조` | `**굵게**` | 포인트 컬러 굵게 |
| `인용` | `>` | 이탤릭/좌측 라인 |
| `코드` | 인라인 코드 | 고정폭(D2Coding) |

> 글머리표(○ / −)는 **문단 모양의 글머리표 기능**으로 스타일에 내장하세요. 본문 글자 앞에 `○`·`-`
> 를 직접 타이핑하지 않습니다(코드가 글자가 아니라 **스타일**로 단계를 구분함).

### 본문 / ○ / − 3단계 구분 (핵심)
보고서 본문은 아래 **3종**으로만 흘러갑니다. 셋의 차이는 **들여쓰기 + 글머리표**입니다.

| 단계 | 스타일 | 글머리표 | 들여쓰기 | Markdown 입력 |
|---|---|---|---|---|
| 평문 | `본문` | 없음 | 0 | 일반 문장(불릿 아님) |
| 1단계 | `동그라미` | ○ | 1수준 | `- 항목` (최상위) |
| 2단계 | `마이너스` | − | 2수준 | `  - 항목` (앞 공백 2칸 = 중첩) |

예시(Markdown → 출력):
```
- (배경) 사업 종료 단계 점검          →  ○ (배경) 사업 종료 단계 점검      [동그라미]
  - 주요 행사 준비 및 참여: ICSE …    →    − 주요 행사 준비 및 참여: ICSE … [마이너스]
  - 사업 모니터링: 성과평가 …          →    − 사업 모니터링: 성과평가 …      [마이너스]
```

### 표(테이블) 스타일
보고서의 표는 **머리행 + 내용행**으로 구성됩니다. 아래 2개 스타일(또는 셀 속성)을 정의합니다.

| 스타일 이름 | 매핑되는 내용 | 권장 서식 |
|---|---|---|
| `표제목` | 표 **머리행** 셀(일자/내용/구분/전문가명/비고 등) | 배경 채움(연회색 또는 메인컬러), **굵게**, **가운데 정렬**, 9~10pt |
| `표본문` | 표 **내용** 셀 (모든 데이터 칸) | 9~10pt, **점(•) 글머리 + 좌측 정렬** 한 가지로 통일 |

- 머리행은 `표제목`, 내용 칸은 `표본문` 으로 통일합니다.
- **표 내용은 복잡하게 나누지 않습니다 — `표본문` 하나(점 + 좌측정렬)면 충분.** 초안 생성이 목적이므로,
  칸별 가운데정렬·색·강조 같은 세부 서식은 **문서가 나온 뒤 한글에서 직접 다듬는** 것을 기본으로 합니다.
- 표 **테두리·열 너비·머리행 반복(쪽 넘김 시)** 은 한글 표 속성으로 양식에 박아두면 유지됩니다(코드가 안 건드림).

- 표지에 `_제목_입력_` 텍스트를 넣어두면 보고서 제목으로 치환됩니다.
- 머리말/꼬리말/로고는 양식에 박아두면 그대로 유지됩니다(코드가 건드리지 않음).
- ⚠️ **양식 본문 영역엔 견본 텍스트를 남기지 마세요**(예: `본문본문본문`). 한컴 OLE 경로는 양식을 열고
  본문을 *이어서* 주입하므로, 남아있는 견본 글자가 결과물에 그대로 찍힙니다. 본문 칸은 **비워** 두세요.
- 완료 후 `assets/hwpx_template.hwpx` 로 저장 → 앱 재시작.

> 📌 **코드 연동 필요(담당 알림)** — 스타일 작업이 끝나면 생성기(`services/studio/generators/hwpx_gen.py`)를 맞춰 드립니다:
> 1. **불릿 단계**: 현재 모든 불릿을 단일 `목록` + "· " 접두로 처리 → `level`(1/2)에 따라 `동그라미`(○)/`마이너스`(−)
>    스타일을 선택하고 글자 접두를 빼 **스타일 글머리표**를 쓰도록.
> 2. **표**: 현재 표를 문단으로 평탄화(`_table_to_blocks`) → 머리행 `표제목`·내용 `표본문`(점+좌측)으로 **진짜 표**(테두리·머리행 반복) 렌더링.
>
> 우선순위는 **"문서가 안정적으로 나오는 것"** 입니다 — 위 매핑이 아직 안 돼도 본문·표는 (평문 형태로라도) 항상 출력되게 둡니다.
> 화려한 서식은 초안이 나온 뒤 한글에서 다듬는 걸 기본으로 합니다.

### HWPX 생성 두 경로 (자동 선택)
1. **한컴 OLE**(권장): Windows + 한컴 한글 + `pywin32`. 양식을 열고 본문을 주입해 `.hwpx` 저장.
2. **ZIP 폴백**: 한글이 없을 때, 위 양식의 **스타일을 재사용**해 본문 섹션만 새로 만들어 `.hwpx` 생성.
3. 둘 다 불가하면 **DOCX/Markdown** 으로 대체 제공(경고로 안내).

> 텍스트 길이가 바뀌면 줄 위치가 약간 어긋날 수 있습니다(한글에서 한 번 열면 자동 재배치).

## PPTX 양식 (슬라이드)
16:9 와이드로 만들고, 슬라이드 마스터에 **두 레이아웃**을 정의합니다.

- **Layout #0 (표지)**: `Title`(발표 제목) + `Subtitle`(부제) placeholder 2개.
- **Layout #1 (본문)**: `Title`(슬라이드 제목) + `Content`(본문) placeholder. 본문에는 **불릿 3~5개**가 자동으로 채워집니다. (다단 아님, 단일 영역)

- 로고/배경/푸터/페이지번호는 마스터에 박아두면 유지됩니다.
- 완료 후 `assets/pptx_template.pptx` 로 저장 → 앱 재시작.

---

## DOCX 양식 (선택)
`assets/docx_template.docx` 에 `제목 1/2/3`, `본문`, `목록` 등 명명 스타일을 정의해 두면
HWPX와 동일한 매핑으로 상속됩니다. 없으면 Word 기본 스타일(Heading 1~3, Normal)로 생성됩니다.
