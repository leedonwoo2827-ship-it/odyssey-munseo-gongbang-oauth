# 00 · 개요 — 문서 생산 스튜디오란?

## 한 줄 요약
입력 문서들을 넣고 **버튼 한 번**으로 보고서·계획서·발표자료 **초안**을 만드는 도구.
(NotebookLM 스튜디오와 같은 사용감)

## 왜 만들었나
AIM 해외사업부 N팀은 해외 ODA 사업 등에서 방문결과·워크숍·SCM·파견계획서 등
**정형 문서를 대량(약 90종, 300여 건)** 으로 만든다. 이전 차수 보고서·현황표·가이드라인을
사람이 일일이 보고 새로 작성하던 일을, 이 도구가 초안까지 자동으로 끌어올린다.

## 핵심 개념 3가지
1. **레시피(유형)** = 산출물 1종의 정의. "필요한 입력 + 작성 지시(프롬프트) + 출력 형식".
   - 스튜디오 화면의 **버튼 하나 = 레시피 하나**. → [04 문서](04_레시피_추가하기.md)
2. **입력(드래그)** = 레시피가 요구하는 문서들(docx/hwpx/xlsx/pptx/pdf)을 끌어 넣기.
3. **지시(채팅)** = 생성 전 메모 + 생성 후 수정 요청. 드래그가 아닌 부분은 모두 채팅으로.

## 동작 흐름
```
[유형 선택] → [입력 끌어넣기 (+ 지시 메모)] → [생성] → [미리보기 + 다운로드] → [채팅으로 수정]
```

## 산출물 형식
- **HWPX**(한글) / **DOCX**(워드) / **PPTX**(슬라이드) / **XLSX**(표) / **MD**(미리보기·백업).
- 회사 **양식(템플릿)** 이 있으면 그 디자인 위에 본문이 들어간다. → [05 문서](05_양식_만들기.md)

## 무엇이 자동이고, 무엇이 사람 몫인가
- 자동: 입력 읽기 → 초안 작성 → 형식 맞춰 파일 생성.
- 사람: 유형 선택, 입력 제공, **사실 검수/최종 편집**. (초안 도구이지 최종본 생성기가 아님)
