# 04 · 새 유형(레시피) 추가하기

**버튼 하나 = 레시피 파일 하나**(`knowledge/recipes/*.yaml`).
엑셀 목록표 16종이 시드로 들어 있고, 나머지 ~90종은 여기에 파일을 더하면 됩니다.

## 방법 A — 파일 편집 (권장, 가장 빠름)
1. `knowledge/recipes/` 에서 비슷한 `.yaml` 을 복사 → 새 이름으로 저장 (예: `17-새보고서.yaml`).
2. 아래 항목을 고치고 저장합니다.
3. 스튜디오 화면을 **새로고침**하면 버튼이 생깁니다. (서버 재시작 불필요)

```yaml
id: 17-새보고서                 # 고유 id (파일명과 같게 권장)
name: 새 보고서                 # 버튼 이름
category: 보고서                # 버튼 묶음
description: 한 줄 설명
model: gemini-3-pro             # (선택) 모델. 비우면 기본값(STUDIO_DEFAULT_MODEL)
output:
  format: hwpx                  # hwpx | docx | pptx | xlsx | md
  template: hwpx_template.hwpx  # (선택) assets/ 의 회사 양식
  filename: 새보고서_{date}      # {date}=오늘 날짜, 확장자는 자동
inputs:
  - { key: inputA, label: "이전 차수 보고서", required: true,  accept: [docx, hwpx, pdf] }
  - { key: inputB, label: "현황표",           required: false, accept: [xlsx] }
prompt: |
  당신은 ... 전문가입니다. 아래 자료로 '새 보고서'를 한국어로 작성하세요.
  {{inputs}}        # 업로드 입력들이 라벨과 함께 자동으로 들어감
  {{instruction}}   # 생성 화면의 추가 지시가 들어감
```

### 항목 설명
| 키 | 뜻 |
|---|---|
| `id` | 고유 식별자. 중복 금지. |
| `name` / `category` | 버튼 표시 이름 / 묶음. |
| `output.format` | 산출물 형식. `pptx`=슬라이드, `xlsx`=표(JSON), 그 외=보고서(Markdown). |
| `output.template` | 회사 양식 파일. 없으면 내장 기본. → [05 문서](05_양식_만들기.md) |
| `inputs[].key` | 내부 식별자(겹치지 않게). `inputA`, `inputB` … 권장. |
| `inputs[].required` | 필수 여부. 첫 입력만 필수로 두는 걸 권장. |
| `inputs[].accept` | 허용 확장자 목록. |
| `prompt` | 작성 지시. `{{inputs}}`, `{{instruction}}` 자리표시자 사용. |

## 방법 B — 앱 내 폼 빌더 (2차 기능)
화면/API로도 추가할 수 있게 백엔드가 열려 있습니다.
`POST /api/studio/recipes` 에 위와 같은 JSON 을 보내면 `.yaml` 파일로 저장됩니다.
(전용 화면은 추후 추가 예정 — 지금은 방법 A 가 가장 쉽습니다.)

## 엑셀에서 한 번에 다시 만들기
목록표를 갱신했다면:
```
python knowledge/recipes/_generate_from_xlsx.py
```
- 기존 `.yaml` 은 **보존**(손본 내용 안전), 없는 항목만 새로 생성합니다.
- 출력 형식은 이름으로 추론합니다(보고서류=hwpx, "Wrap Up/발표"=pptx). 필요하면 `format` 만 고치세요.

## 좋은 프롬프트 팁
- 역할 부여("당신은 …전문가"), 근거 제한("입력 자료에 근거"), 불확실 표시("[확인 필요]"),
  문체/분량 지정. 자주 쓰는 지시는 `prompt` 에 박아두면 매번 안 적어도 됩니다.
